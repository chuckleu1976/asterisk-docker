## Problems Fixed

### 1. SMS Retransmission (RP-ACK EADDRINUSE)
**Root Cause:**
  Asterisk's res_pjsip_messaging.c (send_rpack / volte_send_rp_data) was failing
  with EADDRINUSE (error 120098) because PJSIP tried to create a new TCP socket
  instead of reusing the established VoLTE TCP connection when sending the RP-ACK.
  T-Mobile's SMS-SC never received the acknowledgment and kept retransmitting the
  same SMS indefinitely.

**Fix (rpack_fix.py):**
  The patch script already existed in the repo (committed 2026-05-31 as fc21b43)
  but was only referenced in a thin Dockerfile.patch overlay — NOT in the main
  Dockerfile that actually builds the image.

  Added to asterisk/Dockerfile:
    COPY rpack_fix.py /tmp/rpack_fix.py
    python3 /tmp/rpack_fix.py &&   (before make -j$(nproc))

  The patch sets pjsip_tpselector to transport_state->volte.transport before
  ast_sip_send_request() in both send_rpack() and volte_send_rp_data(), so
  RP-ACK reuses the established TCP connection.

**Commit:** 1111901  "Dockerfile: integrate rpack_fix.py to fix RP-ACK EADDRINUSE on SMS delivery"

### 2. SMS Database Consolidation (previous session, finalised today)
**Change:**
  Three separate SQLite databases merged into one shared sim_inventory.db.
  Added sms table alongside existing sims table.
  Both asterisk and asterisk2 containers mount ./sim_inventory.db:/data/sim_inventory.db:Z
  WAL mode + timeout=10 for safe concurrent writes.

  sms table schema:
    id INTEGER PRIMARY KEY AUTOINCREMENT
    instance TEXT          -- hostname (asterisk1 / asterisk2)
    sender TEXT
    body TEXT
    otp TEXT               -- extracted OTP if present
    received_at DATETIME DEFAULT CURRENT_TIMESTAMP

  compose.yaml changes:
    hostname: asterisk1 / asterisk2
    volumes: ./sim_inventory.db:/data/sim_inventory.db:Z

**Commit:** 6009f72  "consolidate: merge sms table into sim_inventory.db, shared across both instances"
