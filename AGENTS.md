# AGENTS.md — Developer Workflow for asterisk-docker

## Context

This project runs 8 asterisk containers, each connected to a sysmocom SIM
reader.  Containers are started selectively based on SIM presence detected by
the `start_sim --setup` / `start_sim --watch` tooling.

**No network access to Docker Hub / ghcr.io** — all images must be built
locally via `./build-images.sh`.

The current user `ht` cannot run `sudo` non-interactively; `docker` group
membership is used instead.

## Quick start

```bash
make setup           # Bootstrap: probe SIMs, generate configs, start containers
make setup-watch     # Bootstrap + watch for SIM hotswap
make watch           # Watch only (assuming containers already running)
make list            # List readers and SIM info
```

## Directory layout

```
asterisk/                   asterisk container build + entrypoint + ami_usim
compose.yaml                generated by start_sim_container_config.py
config/<N>/                 per-instance config (generated from config/example/)
pcsc-libs/                  custom sysmocom libpcsclite (protocol 4:4)
pcscd/                      pcscd container build
scripts/sim_db.py           SQLite inventory operations
scripts/sim_config_gen.py   config/compose.yaml generation
scripts/sim_docker.py       docker helpers, SIM APDU reading, config updaters
start_sim_container_config.py  CLI entry point
start_sim                   wrapper that activates pysim venv
```

## Key constraints

- **No `sudo -S`**: all docker commands run via the `docker` socket group.
- **No network**: `build-images.sh` requires local pre-pulled images.
- **PID namespace**: all asterisk containers use `pid: service:pcscd` for PC/SC
  socket access.  `docker compose stop` leaves defunct processes in the shared
  PID namespace; use `make clean` to fully tear down.

## Module convention

The 1228-line `start_sim_container_config.py` was split into `scripts/`:

| Module | Responsibility |
|--------|---------------|
| `sim_db.py` | SQLite `readers` + `sims` tables |
| `sim_config_gen.py` | Template config files + `compose.yaml` generation |
| `sim_docker.py` | `docker compose` helpers, APDU SIM reader, config updaters |
| `start_sim_container_config.py` | CLI, setup, watch loop (imports from above) |

## Linting

```bash
ruff check scripts/ start_sim_container_config.py
mypy scripts/ start_sim_container_config.py --ignore-missing-imports
```

## Commit checklist

1. `git status` — only intended files
2. `git diff --cached` — no secrets (IMSI, MSISDN, IMEI)
3. `python3 -c "from scripts.sim_docker import *"` — imports OK
4. Commit subject under 72 chars
